Type Safety — String vs Enum
Catch invalid values at compile time, not in production

String fatigueLevel

"FRESH" "TIRED" "TRED" "" "maybe"

All compile. All valid. Bugs hide until runtime.

➔

FatigueLevel fatigueLevel

FRESH MODERATE TIRED EXHAUSTED

Only 4 valid values. Typos don't compile.

String injuryRisk

"LOW" "MEDIUM" "HIGH" "maybe high"

➔

InjuryRisk injuryRisk

LOW MEDIUM HIGH

String intensity

"LOW" "MEDIUM" "HIGH" "very high"

➔

TrainingIntensity intensity

LOW MEDIUM HIGH

String: "TRED" compiles silently
String: no IDE autocomplete
String: switch can miss cases
Enum: compile-time safety
Enum: IDE autocomplete
Enum: GraphQL validates input
Enum: switch exhaustiveness
Neo4j compatibility: SDN stores enums as their .name() string. FatigueLevel.FRESH → "FRESH" in DB. Existing data works without migration.
