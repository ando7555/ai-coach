PlayerMatchStatService.java — calculating player stats

✗ Pre-Java 8: For Loops

Manual iteration, manual accumulation
int totalGoals = 0; for (PlayerMatchStat s : stats) { totalGoals += s.getGoals(); } int totalAssists = 0; for (PlayerMatchStat s : stats) { totalAssists += s.getAssists(); } List<PlayerMatchStat> rated = new ArrayList<>(); for (PlayerMatchStat s : stats) { if (s.getRating() != null) { rated.add(s); } } double sum = 0.0; for (PlayerMatchStat s : rated) { sum += s.getRating(); } double avg = rated.isEmpty() ? 0.0 : sum / rated.size();
~20 lines of boilerplate
➔

✓ Java 8+: Streams

Declarative, composable, readable
int totalGoals = stats.stream() .mapToInt(PlayerMatchStat::getGoals) .sum(); int totalAssists = stats.stream() .mapToInt(PlayerMatchStat::getAssists) .sum(); var ratedStats = stats.stream() .filter(s -> s.getRating() != null) .toList(); double avg = ratedStats.stream() .mapToDouble( PlayerMatchStat::getRating) .average() .orElse(0.0);
~10 lines — declarative intent

Java 8 to 17 — The Modern Features That Changed Everything

I'm building an AI-powered football coaching app called AI Coach. It runs on Java 17, and virtually every file uses features that didn't exist before Java 8. Lambdas, streams, records, text blocks, pattern matching — these aren't curiosities. They're the backbone of how modern Java code reads and works.

Java's transformation happened in three waves. Here's what each one contributed to the codebase.

Wave 1: Java 8 — The Functional Revolution (2014)

This single release changed more about how Java code looks than everything before it combined:

  • Lambdas — csrf -> csrf.disable() in SecurityConfig replaced 15+ anonymous inner classes with one-liners across the entire codebase.
  • Streams — stats.stream().mapToInt(PlayerMatchStat::getGoals).sum() turns a for-loop with manual accumulation into a single declarative pipeline.
  • Optional — .orElseThrow(() -> new EntityNotFoundException(...)) makes the happy-path vs error-path explicit in the type system. No more null checks.
  • Method references — PlayerMatchStat::getGoals is shorter and more intentional than s -> s.getGoals().

Wave 2: Java 9-11 — Quality of Life (2017-2018)

Small features that reduce ceremony:

  • Immutable factories — Set.of("LOW", "MEDIUM", "HIGH") replaces three lines of Collections.unmodifiableSet(new HashSet<>(...)).
  • var — var auth = new UsernamePasswordAuthenticationToken(...) lets the compiler infer the type when it's obvious from the right-hand side.
  • String.strip() — Unicode-aware whitespace removal for processing AI-generated text that might contain non-standard whitespace characters.

Wave 3: Java 14-17 — Modern Data Modeling (2020-2021)

Java finally got concise data types and modern string handling:

  • Records — 10+ records in the codebase: AuthPayload, MatchAnalysisInput, PlayerPerformanceTrend, and more. Each one replaces 40+ lines of class boilerplate with 1-4 lines.
  • Pattern matching instanceof — 4 patterns in GraphQLExceptionHandler that check the type and bind the variable in one expression, eliminating manual casting.
  • Text blocks — 4 AI prompts in CoachService and RecommendationService use """ blocks paired with .formatted() — the prompt reads exactly as the AI receives it.
  • Stream.toList() — 5 usages replacing .collect(Collectors.toList()) with a shorter, immutable-by-default alternative.
  • Switch expressions — return switch (fatigueLevel) { case "EXHAUSTED" -> "HIGH"; ... }; in CoachService replaces if-else chains with a value-returning expression.
  • Sealed classes — sealed class CoachException permits EntityNotFoundException, AiGenerationException restricts the exception hierarchy to exactly two permitted subtypes, documented and enforced by the compiler.

The Practical Takeaway

Modern Java isn't the same language that earned its "verbose" reputation. Every record in AI Coach would have been 40+ lines as a class. Every stream pipeline would have been a manual loop. Every text block would have been a concatenation nightmare. Every pattern match would have been a check-then-cast.

The bottom line: Java kept the safety — type checking, compile-time errors, the JVM. It shed the ceremony — lambdas for anonymous classes, records for POJOs, text blocks for string concatenation, pattern matching for manual casts. The result is a language that's both safe and expressive.
